1.如何设置Linux系统的密码策略

​ 虽然Linux的设计是安全的,但还是存在许多安全漏洞的风险, 弱密码就是其中之一。 作为系统管理员, 你必须为用户提供一个强密码。因为大部分的系统漏洞就是由于弱密 码而引发的。本教程描述了在基于DEB系统的Linux，比如Debian、Ubuntu、Linux Mint 等和基于RPM系统的Linux ,比如RHEL、CentOS、Scientific Linux等的系统下设 置像密码长度、密码复杂度、密码有效期等密码策略。默认情况下,所有的Linux操作 系统要求用户密码长度最少8个字符。建议不要低于这个限制。并且不要使用你的真 实名称、父母、配偶、孩子的名字,或者你的生日作为密码。即便是一个黑客新手,也可 以很快地破解这类密码。一个好的密码必须是至少8个字符, 并且包含数字、大写字母和特殊符号。

●在基于DEB的系统

通常地，在基于DEB的操作系统中(如ubuntu、debian、Kali ),密码和身份认证相关的 配置文件被存储在/etc/pam.d/目录中。

➢设置最小密码长度

1)编辑/etc/pam.d/common-password文件;

2)找到下面这行:

3)在末尾添加额外的文字:minlen=8。在这里设置的最小密码长度为8。

保存并关闭该文件。这样一来,用户现在不能设置小于8个字符的密码。

➢设置密码复杂度

此设置会强制要求密码中应该包含多少类型,比如大写字母、小写字母和其他字符。

1)用下面命令安装密码质量检测库:

apt-get install libpam-pwquality或apt-get install libpam-cracklib

2)编辑/etc/pam.d/common-password文件:

3)设置密码中至少有一个大写字母,则在下面这行的末尾添加文字ucredit=-1。

4)设置密码中至少有一个小写字母,则在下面这行的末尾添加文字lcredit=-1。

5)设置密码中至少有一个数字,则在下面这行的末尾添加文字dcredit=-1。

6)设置密码中至少有一个特殊字符,则在下面这行的末尾 添加文字ocredit=-1。

7)设置允许输入三次,则在下面这行的末尾添加文字retry=3。

8)设置新密码与旧密码不同的个数为三个，则在下面这行的末尾添加文字difok=3。

9)还可以设置密码中至少含有两种字符的组合，则在下面这行的末尾添加文字minclass=2

➢设置密码有效期

1)编辑/etc/login.defs

2)密码被使用的最长天数，应设置PASS_MAX_DAYS 99999

3)密码更改允许的最小间隔天数,应设置PASS_MIN_DAYS 0

4)密码到期之前发出警告的天数,应设置PASS_WARN_AGE 7

请注意,这些设置将会在新创建的用户中有效。

为已存在的用户设置修改密码有效期,你必须要运行下面的命令:

chage

设置修改密码的最大间隔天数,执行：

$ sudo chage -M <days> <username>

设置修改密码的最小间隔天数,执行：

$ sudo chage -m <days> <username>

设置密码到期之前的警告,执行：

$ sudo chage -W <days> <username>

显示已存在用户的密码,执行：

$ sudo chage -l scst

[例]

设置用户scst 的密码期限是2020年7月21。并且修改密码的最小间隔时间为5天, 最大间隔时间为90天。用户账号将会在10天后被自动锁定, 而且在到期之前的10天 前显示警告信息。

$ sudo chage -E 2020-07-21 -m 5 -M 90 -I 10 -W 10 scst
